# Deep Breach — P1.10 Internal Playtest Checklist

Structured checklist for the internal playtest build covering the story spine through **Engineering Stress Response (P1.9)**. Estimated first-run session: **2.5–3.5 hours** without QA fast-forward.

---

## Setup

1. Clone the repo and install dependencies:
   ```bash
   npm install
   ```
2. Start the app:
   ```bash
   npm start
   ```
3. **New game recommended** for first-run validation. From Base → Settings → Erase Local Save if continuing an old profile.
4. Developer validation (optional):
   ```bash
   npm run check
   ```

**Branch:** `feature/p1.10-playtest-hardening` (or merged main after PR)  
**Save version:** v6 — local only, no cloud sync.

---

## Recommended test device

| Platform | Notes |
|----------|-------|
| **Android emulator** (API 33+) | Primary recommendation — matches most Expo dev workflows |
| **Physical Android device** | Best for touch targets, performance, and pause/resume |
| **iOS Simulator** | Secondary — verify layout if iOS build is in scope |
| **Web (Expo)** | Fast iteration; acceptable for story flow, not final visual sign-off |

Use one **fresh save** and one **mid-spine save** (via QA helper) per platform if time allows.

---

## First-run checklist

- [ ] New game starts at Base with Experimental Trials available
- [ ] Mission Select shows clear lock reasons (not generic errors)
- [ ] Settings → Campaigns opens without crash
- [ ] Captain's Log loads without raw event dumps
- [ ] No user-visible "coming soon" or "not implemented" copy on main paths

---

## Main story path checklist

| Step | Location | Action | Expected outcome |
|------|----------|--------|------------------|
| 1 | Mission Select | Complete all **Experimental Trials** | Trials certified; Operational Integration unlocks |
| 2 | Operational assignments | **Operational Integration** → Acknowledge | Navy integration logged |
| 3 | Operational assignments | **Launch recon** — Operation Dead Beacon | Dive starts; recon debrief on success |
| 4 | Mission Result / debrief | **Dead Beacon data decision** (any option) | Flags granted once; Return dive unlocks |
| 5 | Operational assignments | **Launch return dive** — Return to DBX-03 Site | First anomaly contact on success |
| 6 | Base | **First Contact Analysis** — pick authorization path | Monitoring prep logged; Growing Ocean unlocks |
| 7 | Operational assignments | **Launch monitoring dive** — Growing Ocean Anomaly | Passive monitoring debrief; Command Pressure unlocks |
| 8 | Base / Mission Select | **Command Pressure** — select strategic posture | One posture flag; Expansion Models unlocks |
| 9 | Base / Mission Select | **Abyssal Expansion Models** — prioritize one model | Model flag + modest rewards once; deployment hold visible |
| 10 | Base / Mission Select | **Engineering Stress Response** — select engineering posture | One engineering flag + modest rewards once; descent hold visible |

After step 10, **Descent Authorization Hold** should show in-world threshold copy — not a blank or dev placeholder.

---

## Decision checklist

Each decision is **single-use**. Verify you cannot resolve twice or farm rewards.

| Decision | Where | Verify |
|----------|-------|--------|
| Dead Beacon data disposition | Mission result / debrief after recon | Return dive stays locked until resolved; re-open does not re-grant flags |
| First Contact Analysis | Story briefing | All three paths advance monitoring prep; cannot re-open as pending |
| Command Pressure | Story briefing / base banner | Exactly one strategic flag; modest scrap/research granted once |
| Abyssal Expansion Models | Story briefing / base banner | Exactly one model priority flag; rewards granted once |
| Engineering Stress Response | Story briefing / base banner | Exactly one engineering posture flag; rewards granted once |

---

## Dive checklist

- [ ] Dead Beacon: reach recon depth threshold, ≥1 scan, surface successfully
- [ ] Return dive: interference zone, anomaly contact scan, surface successfully
- [ ] Growing Ocean: passive monitoring drift criteria met on success
- [ ] Engine heat visible in Engineering during dives
- [ ] Failed dives do **not** advance spine events incorrectly
- [ ] Abort / return-to-base does not skip mandatory decisions

---

## Save / load checklist

- [ ] Force-close app mid-dive → relaunch → save restores or fails gracefully (no crash)
- [ ] Complete a decision → kill app → reload → decision remains resolved (not pending again)
- [ ] Mid-spine save (after P1.4) loads with correct Mission Select locks
- [ ] Settings → Erase Local Save resets to new game

---

## Offline / pause behavior

- [ ] Background app during dive → return → dive state consistent
- [ ] No duplicate offline reports after resume
- [ ] Pending mission result persists until acknowledged
