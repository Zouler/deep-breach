# Deep Breach — P1.5 External Playtest Checklist

Use this document to run a structured playtest of the current story spine (Experimental Trials through First Contact Aftermath). Estimated session: **90–150 minutes** for a first-time tester.

---

## Setup

1. Clone the repo and install dependencies:
   ```bash
   npm install
   ```
2. Start the app:
   ```bash
   npm start
   ```
   Open on web, iOS, or Android per your platform.
3. **New game recommended.** From Base → Settings, use reset if continuing an old save.
4. Developer validation (optional):
   ```bash
   npm run check
   ```

**Save version:** v6 — no cloud sync; progress is local only.

## Intended player path (full spine)

| Step | Location | Action | Expected outcome |
|------|----------|--------|------------------|
| 1 | Mission Select | Complete **Pressure Trial I** | Trial certified; next trial unlocks |
| 2 | Mission Select | Complete **Recovery Drill** | Second trial certified |
| 3 | Mission Select | Complete **Signal Calibration** | All Experimental Trials complete |
| 4 | Operational assignments | Open **Operational Integration** → Acknowledge | Navy integration logged |
| 5 | Operational assignments | **Launch recon** — Operation Dead Beacon | Dive starts (~900m, 6 min) |
| 6 | Dive | Reach ~85% target depth, perform ≥1 scan, surface successfully | Recon debrief on mission result |
| 7 | Mission Result | Choose **data disposition** (any option) | Return to Base blocked until resolved |
| 8 | Base / Mission Select | **Launch return dive** — Return to DBX-03 Site | Dive starts (~1200m, 7 min, higher risk) |
| 9 | Dive | Enter interference (~55% depth), scan in contact zone, reach ~85% depth, surface | First contact debrief |
| 10 | Base | Open **First Contact Analysis** | Review findings + authorize next step |
| 11 | Mission Select | **Launch monitoring dive** — Growing Ocean Anomaly | Passive monitoring debrief; spine `growing_ocean_anomaly` on success |

### QA fast-forward (development builds only)

To skip the full spine for P1.6 testing: **Settings → Developer / QA → QA: Advance to Growing Ocean monitoring ready**. See [docs/dev/qa-fast-forward.md](../dev/qa-fast-forward.md).

There is **no debug skip** to this path. A future fast-forward helper may be added for internal QA.

## What to observe

### Clarity
- Are lock reasons on Mission Select understandable?
- Does the Dead Beacon data decision feel required and clear?
- Does mission result explain what to do next after first contact?
- Is the Growing Ocean Anomaly card clearly “not yet playable”?

### Balance
- **Trials:** Do early trials teach systems without feeling unfair?
- **Dead Beacon:** Tense but completable on first attempt?
- **Return dive:** Noticeably harder than Dead Beacon, but not random death?
- **Engine Heat:** Pressure in Engine Bay without dominating every mission?
- **Anomaly interference:** Noticeable sensor events and hull stress, predictable timing?

### Flow blockers
- Any button that does nothing or misleads?
- Can you skip the Dead Beacon data decision?
- Can you launch Return before P1.2 resolution?
- Can you complete First Contact Analysis from a new game?

### Tone
- Hard sci-fi, restrained, bureaucratic — **no** alien/monster/supernatural reveal yet.

---

## Feedback questions

1. At what step did you first feel confused about what to do next?
2. Which mission felt too easy, too hard, or unfair — and why?
3. Did Engine Heat feel like a meaningful mechanic or noise?
4. Did anomaly interference during Return feel intentional or random?
5. Was the Dead Beacon data decision meaningful, or did all options feel the same?
6. After First Contact Analysis, do you understand that the next stage is **preparation**, not explanation?
7. Any copy that felt like placeholder, WIP, or developer text?
8. Would you continue playing after this session? What would you expect next?
