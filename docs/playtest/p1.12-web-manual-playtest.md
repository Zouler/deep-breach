## Executive summary

P1.12 performed a **real interactive web playtest** against the live Expo bundle at `http://localhost:8081`. The **cursor-ide-browser MCP was unavailable** in this agent environment (`browser_navigate` returned *"No browser tab available"* on every attempt). Playwright Chromium was used as a fallback for genuine browser interaction (clicks, navigation, `localStorage` inspection, page reload).

**Fresh-save entry** (Start New Expedition → Skip Briefing → trials/mission select) works on web. **Live save/load roundtrip** is **verified for early spine checkpoints** on web via `localStorage` (`@deep_breach/game_state_v1`) + full page reload. **Mid-spine QA fast-forward helpers (Growing Ocean and later) do not advance state on web** — they stall after the Return to DBX-03 dive simulation, blocking automated save/load verification for later checkpoints in this session.

**M-02:** Partially closed on web for early checkpoints; mid-spine live roundtrip remains open pending QA-helper web fix or manual decision UI walkthrough.

**Recommendation:** Progression sign-off for P1.0–P1.2 early spine on web; investigate QA helper web regression before relying on web for P1.4+ save/load QA. Physical Android pass still recommended for touch targets and native AsyncStorage semantics.
